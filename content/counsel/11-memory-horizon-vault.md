# Official naming
- **Memory Horizon** — the three-layer local memory model on the Sovereign brain.
- Layers: **Hot Cache** → **Session Recall** → **Vault** (optional).
- Never say “we store your memory in our cloud.” Never claim Vault is on by default.

# One-sentence truth
Your Sovereign remembers on hardware you own: a short always-on card, searchable past chats, and — only if you choose — a sealed Vault for long-horizon notes.

# Talk track (buyers)
1. Most owners need only Hot Cache + Session Recall — nothing special to configure.
2. Vault is opt-in encrypted archive: propose → commit (not transcript dumps).
3. Unlock is deliberate; locked Vault protects a stolen disk, not malware while unlocked.
4. IT: Phase 3 owner-only Vault after Remote is stable — see /security and /security#faq.
5. Help: https://help.1human1ai.com/docs/privacy-and-security/memory-horizon-and-vault

# What it is / isn’t
**Is:** local Memory Horizon on the brain; Vault as sealed far horizon under owner passphrase.

**Is not:** Centurion Limited cloud backup; automatic ingestion of every chat; the same thing as Collaborations (mission cockpits ≠ memory archive).

# Objection bridges
- “Where does memory live?” → On your Sovereign. Layers are local. Link help Memory Horizon & Vault.
- “Do you train on our notes?” → No product telemetry from the private world; Vault stays on the brain.
- “Can IT roll this out safely?” → Owner first; Vault optional and late — Security brief + FAQ.

# Preferred phrasing
- “Memory Horizon — Hot Cache, Session Recall, optional Vault.”
- “Vault is sealed on your brain — propose, then commit.”
Avoid: swarm memory, cloud vault, YOLO memory, SOUL.md as a customer term.
